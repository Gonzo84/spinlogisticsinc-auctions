## Fix Details

### BUG-D01: Seller settlements page shows no data after payment settlement

**Root Cause:** The committed version of `PaymentEventSellerConsumer.kt` only handled `payment.settlement.ready` (creating settlement rows) and `payment.checkout.completed` events. It was missing a handler for `payment.settlement.settled` events. Additionally, settlement rows were inserted without a `paymentId`, making it impossible to locate them when the settled event arrived.

**Fix (3 files):**

1. **`services/seller-service/src/main/kotlin/.../nats/PaymentEventSellerConsumer.kt`**
   - Added `SUBJECT_SETTLEMENT_SETTLED` constant
   - Added routing for `payment.settlement.settled` subject in `handleMessage()`
   - Added `handleSettlementSettled()` method that updates settlement status from READY to PAID via paymentId lookup
   - Modified `handleSettlementReady()` to extract and pass `paymentId` to the settlement insert

2. **`services/seller-service/src/main/kotlin/.../repository/SellerProfileRepository.kt`**
   - Added `payment_id` column to `INSERT_SETTLEMENT` SQL
   - Added `UPDATE_SETTLEMENT_STATUS_BY_PAYMENT_ID` SQL constant
   - Added `paymentId` parameter to `insertSettlement()` method
   - Added `updateSettlementStatusByPaymentId()` method

3. **`services/seller-service/src/main/resources/db/migration/V003__add_payment_id_to_settlements.sql`** (new)
   - Adds `payment_id UUID` column to `app.seller_settlements`
   - Adds index on `payment_id` for lookup performance
